## Strategic Discussion: Why the Experiment Order Changed

### The Core Problem

DRE (replay + EWC + snapshot self-distillation) is the current best method. The snapshot teacher is 100% accurate on old tasks by construction. GRACE's MedSAM3 adapter is ~56.5% accurate. For GRACE to win head-to-head on logit-level KD, foundation features need to provide enough transfer benefit to overcome a ~44% accuracy gap. Estimated probability: ~30%.

### Options Considered

**Option A: GRACE vs DRE head-to-head (original plan)**
- Pro: Clean story if it works
- Con: ~30% chance of working; if it fails, no paper
- Verdict: Too risky as the ONLY experiment

**Option B: Hybrid DRE + GRACE (combining snapshot + foundation KD)**
- Pro: Foundation teacher can only help, never hurt; ~70% publishable
- Con: "Kitchen sink" risk — 6 loss terms to beat a 3-term baseline. Ablation table may show GRACE adds only marginal improvement. Reviewers will see through it.
- Con: Feature-level KD across architectures (ViT -> UNet) requires a learned projection that itself needs CL management. High technical risk.
- Con: Hybrid undermines the crossover argument (snapshot still present prevents degradation, so GRACE's theoretical advantage over long sequences may not manifest clearly)
- Verdict: Too complex for first experiment. Save as fallback.

**Option C: GRACE standalone on 5-task sequence (Codex's recommendation)**
- Pro: Proposition 2 predicts GRACE advantage grows with sequence length. On 5 tasks, snapshot from Task A is 4 tasks stale. GRACE's external teacher never degrades.
- Pro: Highest-information experiment — if GRACE wins here, paper is done without hybrid complexity
- Pro: Only 3 GPU-days
- Con: If GRACE still loses on 5 tasks, the method paper is dead
- Verdict: **Run this first.** It determines everything.

**Option D: DRE + Prototype Regularization only (simpler hybrid)**
- Pro: One new component (CPA from MedSAM3 features), one hyperparameter, clean ablation
- Pro: Prototypes use frozen backbone features directly — no adapter bottleneck, no projection needed
- Pro: ~65-70% publishable
- Con: Smaller claim than full GRACE
- Verdict: **Run as Step 2 if Step 1 fails.** Still a publishable paper.

### Decision: Sequential Experiments, Each Informing the Next

Run experiments in this order. Stop as soon as you have a positive result.

### Step 1: GRACE Standalone on 5-Task ABCDE (THE CRITICAL EXPERIMENT)

**Why this first:** Proposition 2 predicts GRACE's advantage grows with sequence length. If GRACE can't beat DRE on 5 tasks (where snapshot degradation should be maximal), it won't beat DRE anywhere. This is the highest-information experiment: it determines whether the method paper is viable.

**What to run:**

Experiment 1A — DRE baseline on ABCDE:
```bash
python scripts/run_continual.py \
  --base-config configs/base.yaml \
  --task-config configs/tasks/totalseg_ABCDE_sequence.yaml \
  --method-config configs/methods/distill_replay_ewc.yaml \
  --dataset-config configs/datasets/totalseg_clean.yaml \
  --override-config configs/overrides/full_50ep.yaml \
  --output-dir outputs/step1_dre_ABCDE_s42
```

Experiment 1B — GRACE standalone on ABCDE:
```bash
python scripts/run_continual.py \
  --base-config configs/base.yaml \
  --task-config configs/tasks/totalseg_ABCDE_sequence.yaml \
  --method-config configs/methods/distill_replay_ewc_medsam3_gated.yaml \
  --dataset-config configs/datasets/totalseg_clean.yaml \
  --override-config configs/overrides/full_50ep.yaml \
  --output-dir outputs/step1_grace_ABCDE_s42
```

**CRITICAL CONFIG CHECKS before running 1B:**
- `kd.weight: 0.1` (NOT 0.7 — must match DRE for fair comparison)
- `adapter_ckpt_path:` points to finished MedSAM3 checkpoint
- `adapter_type: gated_residual`
- Resolution: 128³ (set by full_50ep.yaml)
- Both use same seed (42), same replay buffer (1000), same EWC weight (0.2)

**Note on adapter pretraining for Tasks B-D:** GRACE needs adapter residuals for each task. The current pretraining script trains for Task A. For the 5-task sequence, new residuals should be added at each task transition via `reconfigure_adapter()` — this happens automatically in `multi_task_trainer.py` via `set_task_output_channels()`. Verify that residuals are actually being added by checking logs for "GatedResidualAdapter: added task" messages.

**Also need:** Pretrain the gated adapter on each task's data, or at minimum on Task A data. The per-task residual training happens automatically during task transition, but the core + gate + initial residual must be pretrained.

**Expected runtime:** ~2-3 GPU-days per experiment (5 tasks x 50 epochs x ~100 steps)

**How to interpret results:**

| Result | Interpretation | Next Step |
|--------|---------------|-----------|
| GRACE beats DRE on mean forgetting across 5 tasks | **Paper works.** | Run ablation + multi-seed (Step 3) |
| GRACE ties on early tasks but wins on later tasks | **Crossover story works.** | Run ablation + multi-seed (Step 3) |
| GRACE ties or loses on all tasks | Method contribution is weak | Go to Step 2 |

### Step 2: DRE + Prototype Regularization Only (FALLBACK)

**Only run if Step 1 fails.**

**Why this:** Prototypes from MedSAM3's frozen backbone feature space provide a signal that snapshot self-distillation cannot: task-agnostic anatomical priors computed in a stable, non-drifting feature space. No learned projection needed. One new hyperparameter. Clean ablation.

**What to run:**

Use the DRE config but add prototype KD from the MedSAM3 teacher:
- `method.kd.prototype.weight: 0.3`
- `method.kd.prototype.temperature: 0.5`
- Teacher config pointing to MedSAM3 with the pretrained gated adapter

This is essentially `distill_replay_ewc_medsam3_gated.yaml` but with snapshot self-distillation ALSO active. To implement:
- In `distill_replay_ewc.py`, the snapshot teacher is created in `post_task_update()` via `self.teacher.snapshot(model)` when `not self.teacher.is_external`
- For the hybrid, we need BOTH: snapshot for replay KD + external teacher for prototype KD
- This may require a small code change: allow the method to hold both a snapshot teacher and an external teacher simultaneously

**If this beats DRE:**
- Paper narrative: "Foundation model features as complementary regularization for continual segmentation"
- Smaller claim than full GRACE, but clean and publishable
- Run ablation: DRE vs DRE+CPA, with and without MedSAM3 features

**If this also fails:** Go to Step 2B.
